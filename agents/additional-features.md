# nugget 추가 기획 제안 v0.1

작성일: 2026-03-06

본 문서는 `implementation-plan.md(v0.2)`를 기준으로, MVP 구현 전에 함께 넣으면 효과가 큰 항목만 정리한 제안서다.

## 1) 즉시 포함 권장 (MVP 범위 안)

- 저장 시점 히스토리
  - 장소 저장 레코드에 `created_at`을 적극 활용해 "최근 저장한 순" 정렬 제공
  - 지도 마커 상세에서 "누가 언제 저장했는지" 함께 노출

- 간단 프로필 카드
  - `팔로잉 수 / 팔로워 수 / 저장 수`를 공통 `UserRow`와 프로필 카드에서 일관 노출
  - 팔로우 목록/검색/지도 상세에서 동일한 사용자 정보 스냅샷 사용

- 지도 성능 기본 최적화
  - 현재 뷰포트 바깥 마커는 렌더 제외
  - 지도 이동이 잦을 때 API 호출 디바운스(예: 300ms)

- 입력 유효성 강화
  - 닉네임: 2~20자, trim 후 공백-only 금지
  - 메모: 500자 제한 + 남은 글자 수 표시
  - 검색어: 1자 이하는 요청 차단

- 공통 에러 처리 규칙
  - API 에러 응답 포맷 통일: `{ code, message }`
  - 네트워크/서버 오류 시 재시도 버튼을 가진 토스트 또는 인라인 에러 노출

## 2) 바로 다음 단계(Phase 2) 권장

- 북마크 메타 확장
  - 저장 장소에 `rating(선택)`, `tags(선택)` 확장 가능 구조로 설계
  - 당장은 UI 숨김, DB/API shape만 확장 가능하게 유지

- 유저 검색 품질 개선
  - 접두사 우선 정렬 + 부분 일치 정렬
  - 본인 계정 검색 결과 제외 옵션

- 색상 일관성
  - 타인 마커 색상은 `user_id hash -> 팔레트 index` 고정 함수로 계산
  - 앱 재실행/기기 변경 후에도 동일 유저 색상 유지

## 3) 추후 기능 후보 (이번 구현 제외)

- 알림
  - 새 팔로워/맞팔/내 장소 저장 반응 알림

- 소셜 확장
  - 공개 프로필 URL 공유
  - 장소 리스트 공유 링크

- 안전장치
  - 유저 차단/신고
  - 비공개 계정 옵션

## 4) 이번 구현에 반영할 결정

- 즉시 포함 권장 항목은 이번 구현에 반영
- Phase 2 항목은 구조만 열어두고, UI는 핵심 기능 안정화 후 추가
- 추후 기능 후보는 백로그로 유지
